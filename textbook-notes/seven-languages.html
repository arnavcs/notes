What is Ruby?

Ruby is an interpreted, object oriented, and dynamically typed language which is duck typed.

Object Oriented

In Ruby, most things are objects, even numbers and classes themselves. For example: 

>> 1.class
=> Integer
>> Integer.class
=> Class
>> Class.class
=> Class

The methods of an object can be returned with the .methods method.

Comments

Comments are denoted with #.

Strings

If strings are surrounded by one quote, they are to be considered literal, but if they are in double quotes, they are evaluated, rplacing anything in #{} with its evaluated value.

Naming Conventions

Class names are usually PascalCase, variable and method names are snake_case, constants are in ALL_CAPS, functions that test a condition end in a “huh” (?) and those that modify the arguments passed in end with a “bang” (!).

Descisions

Descisions can be made with the if[,elsif,else,end], unless[,else,end], while[,end], and unless[,end] keywords inline or in blocks. These are special keywords that do not seem to be objects or methods. Additionally, anything other than nil and false evaluates to true in these conditionals. 

puts 'x != 1' unless x == 1

if x == 1
    puts 'x == 1'
end

x = x + 1 while x < 10

Logical operations like && and || are paired with & and |, variants of the two which do not short-circuit.

Duck Typing

Being strongly typed, Ruby does a type check (at run time since it is dynamically typed) to check type collisions. As such, you don't have to inherit from the same parent to be used in the same way.

Symbols

Ruby symbols are identifiers, each starting with :. The advantage of symbols over strings is that strings with the same value can be different physical strings, but each symbol is only its value. This is to say, that the .object_id method will always return the same.

Functions

A function can be defined with def and can have parameters passed into it. For example, the following shows an implicit return. The function is also an object. As you can see, parenthesis are implicit for the last parameter of a function. 

>> def succ n
>>   n + 1
>> end
=> :succ
>> succ 5 * 2
=> 11

Arrays

As a result of the dynamic typing system, Ruby arrays do not have to be homogenous. Arrays can be used as queues, stacks, or even sets. The two methods [] and []= allow access to the array, and there is syntactic sugar for them which makes it resemble C array acess. For example: 

>> [0, 1].class
=> Array
>> [2, 3][1]
=> 3

Hashes

Hashes follow the syntax below, and are often useful for passing into functions when named parameters are desired. Again, because of the dynamic typing system, hashes don't have to be homogenous. 

>> person = {:height => 3, :name => 'anon'}
>> person[:height]
=> 3
>> person[:weight]
=> nil

Code Blocks and Yield

A code block is denoted with either { and } or do and end. It is essentially a function without a name, and as such, can also have a parameters. For example, the Integer times method and the Array each methods can take in a code block. 

>> [1, 3, 2].sort {|x, y| x <=> y}
=> [1, 2, 3]

The yield function is useful for making your own functions which take a block. For example, if you wanted to make our own times method: 

class Fixnum
    def my_times
        for i in 0..(self-1) do
            yield i
        end
    end
end

Blocks can also be passed as first-class parameters by using & in the parameter name and can be called with the .call method.

Blocks are very versatile and can be used to specify that certain code should be delayed in execution, condiditionally executed, or to enfore transaction policy.

Classes

Ruby classes can only inherit from one parent class, the superclass. Everything eventually inherets from the Object class (or in Ruby 3, the BasicObject class). Even classes themselves are instances of the Class class, whose superclass is Module, whose superclass is Object.

You can open up an existing class and add methods to it as you would define a class. Additionally, instance variables (one per object) are prepended with @ and class variables (one per class) with @@. The keywords attr (and attr_accessor) define an instance variable, an accessor, (and a setter). 

class Tree
    attr_accessor :children, :node_name

    def initialize(name, children=[])
        @children = children
        @node_name = name
    end
    
    def visit_all(&block)
        visit &block
        children.each {|c| c.visit_all &block}
    end

    def visit(&block)
        block.call self
    end
end
